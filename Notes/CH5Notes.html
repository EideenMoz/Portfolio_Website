flex-direction: row - Default value - The main start and main end depend on the writing mode - Flex items are the same height and width as their content flex-direction: column - Changes the main axis to a vertical direction - The main start and main end run from top to bottom - Flex items span the width of their container row-reverse, column-reverse - reverses the direction of the main axis - bottom to top, and right to left flex-wrap - Flex items are arranged on a single axis by default, as defined by the flex-wrap property flex-wrap: nowrap - Defaul value - flex items stay on the same line - Remaining space in container will be displayed - Flex items will shrink when the items are larger than the container - Content will overflow if conatiner is too small flex-wrap: wrap - Flex items that don't fit will wrap to the next line - Remaining space in container will be displayed flex-wrap: wrap-reverse - Changes the direction of the wrapped items - Only the cross start and cross end are reversed, not the main start and main end Shorthand: flex-flow - Longhand flex-direction: column; flex-wrap: wrap; - Shorthand flex-flow: column wrap; ------------------------------------------------------------------------------------------------------------------------------------ Sizing with flex - Used to size flex items relative to their container i.e. /* Shorthand */ flex: 0 1 auto; /* Longhand */ flex-grow: 0; flex-shrink: 1; flex-basis: auto; flex-grow - Determines how items expand in the flex container - Defined by a unitless value - Default value of 0, meaning the flex items won't expand and the flex container won't be filled i.e. flex-grow: 0; - A value of 1 will expand all the flex items equally, filling the flex container i.e. flex-grow: 1; - The extra space is divided into equal parts, then added to the current size of each flex item flex-shrink - Controls how flex items will shrink if they are larger than the container - Defined with a unitless value - Default value of 1, all flex items will shrink by the same amount i.e. flex-shrink: 1; - Value of 0 will prevent the flex items from shrinking at all flex-basis - Sets the initial size of flex items - Defined with , , or keyword values - Defaul value is auto - Flex items are the same size as their content - box-sizing: content-box, meaning any padding or border will make the flex item larger flex - The shorthand syntax follows a specific order i.e. /* flex-grow | flex-shrink | flex-basis */ flex: 0 1 auto; Putting it together i.e. .flex-container { background: powderblue; border: 5px solid darkslategrey display: flex; flex-wrap: wrap; flex-direction: column; } .flex-items { /* flex-grow | flex-shrink | flex-basis */ flex: 1 1 auto; border: 4px solid white; padding: 10px; box-sizing: border-box; } .item-1 { flex-grow: 2; }

Define the Grid Container - Use the display property with grid or inline-grid - The grid items will display in a single column i.e. .grid-container { display: grid; /* OR */ display: inline-grid; } - When using the value of grid, the width of the grid items will span the width of the container - The container will be displayed as a block element and stack on top of other block elements i.e. .grid-container { display: grid; } - When using the value of inline-grid, the width of the grid items will span the width of their content - The container will be displayed inline next to other inline elements i.e. .grid-container { display: inline-grid; } - Grid lines divide the grid into columns and rows - Grid lines also determine the positiion of grid items using a numerical index or custom name - A negative numerical index can be used to reference the opposite end of the grid - A grid cell is a single unit where a grid row and column intersect - A grid track refers to the columns and rows - Grid tracks can also be separated by a gutter - Grid areas may span one or more grid cells and grid tracks - Unlink grid tracks, this can span across rows and columns ------------------------------------------------------------------------------------------------------------------------------------ Creating a Grid - Grid tracks are defined in the grid container with grid-template-columns and grid-template-rows - These properties are declared in the grid container - This will create an explicit grid i.e. .grid-container { display: grid; grid-template-columns: 25% 50% 25%; /* Three columns /* grid-template-rows: 100px 100px; /* Two rows */ } The Fraction Unit: fr - Represents a fraction of the availbale space in the grid container i.e. grid-template-columns: 1fr 1fr 1fr; /* Distributes the columns evenly with the same width */ i.e. grid-template-columns: 1fr 2fr 1fr; /* The second track takes up twice the amount of space as the first and second tracks */ The repeat() Function i.e. /* Synatx */ repeat(number of tracks, size of tracks) grid-template-columns: 1fr 1fr 1fr; /* Same as */ grid-template-columns: repeat(3, 1fr); i.e. grid-template-columns: 50px repeat(2, 1fr); i.e.
